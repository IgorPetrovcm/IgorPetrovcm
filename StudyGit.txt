Создание рабочей области:
->  clone  - Клонирование репозитория в новый католог
    init   - Создание пустого репозитория Git

Работа с текущими изменениями
->  add    - Добавление содержимого файла в индекс
    mv     - Перемещение или переименование файла, каталога
             или символьной ссылки
    restore- Восстановление файлов в рабочем каталоге
    rm     - Удаление файлов из рабочего каталога и индекса

Просмотр истории и текущего состояния
->  bisect - Выполнение двоичного поика коммита, который
             вносит ошибку
    diff   - Вывод разницы между коммитами, коммитом и
             рабочим каталогом
    grep   - Вывод строк, соответствующих шаблону
    log    - Вывод истории коммитов
    show   - вывод различных типов объектов
    status - Вывод состояния рабочего каталога

Выращивание, маркировка и правка вашей общей истории
->  branch - Вывод списка, создание или удаление веток
    commit - Запись изменений в репозиторий
    merge  - Объединение одной ил нескольких историй
             разработки вместе
    rebase - Повторное применение коммитов над верхушкой
             другой ветки
    reset  - Сброс текущего состояния HEAD на указанное
             состояние
    switch - Переключение веток
    tag    - Создание, вывод списка, удаление или проверка
             метки, подписаннйо с помощью GPG
Совместная работа
    fetch  - Загрузка объктов и ссылок из другого репозитория
    pull   - Извлечение изменений и объединение с другими
    push   - Обновление внешних ссылок и связаннох объектов
